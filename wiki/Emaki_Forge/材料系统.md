# Emaki Forge - 材料系统

## 概述

材料（ForgeMaterial）是锻造系统的核心输入，定义了锻造所需的物品及其属性贡献。材料系统支持多种效果类型，可以为锻造产物添加属性、修改名称和描述。

---

## 材料配置

### 配置文件位置

```
plugins/Emaki_Forge/materials/
├── iron_essence.yml      # 铁精粹
├── flame_crystal.yml     # 火焰结晶

├── xuan_tie_core.yml     # 玄铁核
└── yun_lei_sha.yml       # 云雷砂
```

### 基础配置结构

```yaml
id: "iron_essence"
display_name: "<white>铁精粹</white>"
description:
  - "<gray>蕴含铁之精华的材料"
  - "<gray>可用于强化武器的基础材料"

item: "ni-material_iron_essence"

capacity_cost:
  type: "constant"
  value: 5

priority:
  type: "constant"
  value: 10

effects:
  - type: "stat_contribution"
    stats:
      physical_damage:
        type: "constant"
        value: 5
      max_health:
        type: "constant"
        value: 2
```

## 配置项详解

### id

**类型**：String  
**必填**：是  
**说明**：材料的唯一标识符

```yaml
id: "iron_essence"
```

---

### display_name

**类型**：String  
**必填**：否（默认使用 id）  
**说明**：材料的显示名称

```yaml
display_name: "<white>铁精粹</white>"
```

---

### description

**类型**：List\<String\>  
**必填**：否  
**说明**：材料的描述文本

```yaml
description:
  - "<gray>蕴含铁之精华的材料"
  - "<gray>可用于强化武器的基础材料"
```

---

### item

**类型**：String（物品源简写）  
**必填**：是  
**说明**：材料对应的物品

```yaml
# NeigeItems 物品
item: "ni-material_iron_essence"

# CraftEngine 物品
item: "ce-materials:iron_essence"

# 原版物品
item: "v-IRON_INGOT"
```

---

### capacity_cost

**类型**：随机配置  
**必填**：否（默认 0）  
**说明**：材料消耗的锻造容量

```yaml
# 固定值
capacity_cost: 5

# 随机值
capacity_cost:
  type: "uniform"
  min: 3
  max: 8

# 表达式
capacity_cost:
  type: "expression"
  expression: "{base} * 1.5"
```

---

### priority

**类型**：随机配置  
**必填**：否（默认 0）  
**说明**：材料的优先级，用于材料匹配排序

```yaml
priority: 10
```

## 效果系统

### stat_contribution - 属性贡献

为锻造产物添加属性值：

```yaml
effects:
  - type: "stat_contribution"
    stats:
      physical_damage:
        type: "constant"
        value: 5
      max_health:
        type: "gaussian"
        mean: 10
        std_dev: 2
        min: 5
        max: 20
      attack_speed:
        type: "uniform"
        min: 0.1
        max: 0.3
```

**支持的随机类型**：
- `constant`：固定值
- `uniform`：均匀分布
- `gaussian`：高斯分布
- `skew_normal`：偏态分布
- `triangle`：三角分布
- `expression`：表达式

---

### name_modify - 名称修改

修改锻造产物的名称：

```yaml
effects:
  - type: "name_modify"
    operation: "append_suffix"
    value: " <red>[强化]</red>"
```

**操作类型**：

| 操作 | 说明 |
|------|------|
| `append_suffix` | 添加后缀 |
| `prepend_prefix` | 添加前缀 |
| `replace` | 替换名称 |

---

### lore_operation - 描述操作

修改锻造产物的描述：

```yaml
effects:
  - type: "lore_operation"
    operations:
      - operation: "insert_below"
        target_pattern: "锻造属性:"
        content:
          - "<gray>物理攻击: <red>+{physical_damage}"
          - "<gray>生命值: <green>+{max_health}"
      - operation: "append_lines"
        content:
          - "<yellow>锻造材料: 铁精粹"
```

**操作类型**：

| 操作 | 说明 |
|------|------|
| `append_lines` | 追加行 |
| `prepend_lines` | 插入行到开头 |
| `insert_below` | 插入到匹配行下方 |
| `insert_above` | 插入到匹配行上方 |
| `replace_line` | 替换匹配行 |
| `remove_lines` | 移除匹配行 |

## 完整配置示例

### 示例1：基础强化材料

```yaml
id: "iron_essence"
display_name: "<white>铁精粹</white>"
description:
  - "<gray>蕴含铁之精华的材料"
  - "<gray>可用于强化武器的基础材料"

item: "ni-material_iron_essence"

capacity_cost: 5

priority: 10

effects:
  - type: "stat_contribution"
    stats:
      physical_damage:
        type: "constant"
        value: 5
      max_health:
        type: "constant"
        value: 2

  - type: "lore_operation"
    operations:
      - operation: "insert_below"
        target_pattern: "锻造属性:"
        content:
          - "<gray>物理攻击: <red>+{physical_damage}"
          - "<gray>生命值: <green>+{max_health}"
```

---

### 示例2：元素材料

```yaml
id: "flame_crystal"
display_name: "<gold>火焰结晶</gold>"
description:
  - "<red>蕴含烈焰之力的结晶"
  - "<gray>可为武器附加火焰属性"

item: "ni-material_flame_crystal"

capacity_cost: 10

priority: 20

effects:
  - type: "stat_contribution"
    stats:
      fire_damage:
        type: "uniform"
        min: 10
        max: 20
      burn_chance:
        type: "constant"
        value: 0.15

  - type: "name_modify"
    operation: "append_suffix"
    value: " <red>[烈焰]</red>"

  - type: "lore_operation"
    operations:
      - operation: "append_lines"
        content:
          - "<red>火焰伤害: +{fire_damage}"
          - "<red>燃烧几率: {burn_chance}%"
```

---

### 示例3：稀有材料

```yaml
id: "dragon_scale"
display_name: "<dark_purple>龙鳞</dark_purple>"
description:
  - "<light_purple>传说中的龙鳞"
  - "<gray>蕴含强大的龙之力"

  type: "gaussian"
  mean: 25
  std_dev: 5
  min: 15
  max: 35

  - type: "stat_contribution"
    stats:
      all_damage:
        type: "skew_normal"
        mean: 15
        std_dev: 3
        skewness: 0.3
        min: 10
        max: 25
      armor:
        type: "constant"
        value: 5
      magic_resist:
        type: "uniform"
        min: 3
        max: 8

  - type: "name_modify"
    operation: "prepend_prefix"
    value: "<dark_purple>龙之</dark_purple> "

  - type: "lore_operation"
    operations:
      - operation: "prepend_lines"
        content:
          - "<dark_purple>═══ 龙之祝福 ═══"
      - operation: "append_lines"
        content:
          - "<dark_purple>龙鳞加持，力量倍增"
```

## 材料匹配规则

### 必需材料

配方中指定的必需材料必须放入：

```yaml
required_materials:
  - id: "iron_essence"
    count: 3
  - id: "flame_crystal"
    count: 1
```

### 可选材料

可选材料受白名单/黑名单限制：

```yaml
optional_materials:
  enabled: true
  whitelist: []           # 空 = 允许所有
  blacklist:
    - "cursed_stone"      # 禁止使用诅咒之石
  max_count: 5            # 最多放入5个可选材料
```

## 属性累加规则

### 同类属性累加

多个材料的相同属性会累加：

```yaml
# 材料1: 铁精粹
stats:
  physical_damage: 5

# 材料2: 火焰结晶
stats:
  physical_damage: 3
  fire_damage: 10

# 最终结果
physical_damage: 8  (5+3)
fire_damage: 10
```

### 优先级影响

高优先级材料的属性修改后执行：

```yaml
# 材料1 (priority: 10)
name_modify: append_suffix " [强化]"

# 材料2 (priority: 20)
name_modify: append_suffix " [精良]"

# 最终名称
"铁剑 [强化] [精良]"
```

## 最佳实践

### 1. 合理设置容量消耗

根据材料稀有度设置：

```yaml
# 普通材料
capacity_cost: 5

# 稀有材料
capacity_cost: 15

# 传说材料
capacity_cost: 30
```

### 2. 使用随机属性增加趣味性

```yaml
stats:
  damage:
    type: "gaussian"
    mean: 10
    std_dev: 2
    min: 5
    max: 15
```

### 3. 提供清晰的属性说明

```yaml
- type: "lore_operation"
  operations:
    - operation: "append_lines"
      content:
        - "<gray>──────────────"
        - "<gold>材料效果:"
        - "<gray>物理攻击: <red>+{physical_damage}"
        - "<gray>生命值: <green>+{max_health}"
```

## 故障排查

### 材料无法识别

**检查项**：
1. `item` 配置的物品是否存在
2. 物品源前缀是否正确
3. 材料ID是否与配方中引用的一致

### 属性未生效

**检查项**：
1. `effects` 配置格式是否正确
2. 属性名是否被产物支持
3. `stat_contribution` 的 stats 配置是否有效

### 容量计算异常

**检查项**：
1. `capacity_cost` 数值是否合理
2. 随机配置是否正确
3. 是否超过图纸容量上限
