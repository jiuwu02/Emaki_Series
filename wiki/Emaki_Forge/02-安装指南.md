# Emaki Forge - 安装指南

## 系统要求

### 必需环境

| 要求项 | 最低版本 | 推荐版本 |
|--------|---------|---------|
| Minecraft Server | 1.21 | 1.21+ |
| 服务端类型 | Paper | Paper (最新版) |
| Java Runtime | 21 | 21+ |
| Emaki CoreLib | 1.0.0 | 最新版 |

### 可选依赖

| 插件 | 版本 | 功能 |
|------|------|------|
| NeigeItems | 1.21.141+ | 自定义物品支持 |
| CraftEngine | 0.0.67+ | 自定义物品支持 |
| PlaceholderAPI | 2.12.2+ | 占位符支持 |

## 安装步骤

### 第一步：安装 Emaki CoreLib

Emaki Forge 依赖 Emaki CoreLib，请先安装：

1. 下载 `Emaki_CoreLib-x.x.x.jar`
2. 放入 `plugins` 目录
3. 启动服务器确认 CoreLib 正常加载

### 第二步：安装 Emaki Forge

1. 下载 `Emaki_Forge-x.x.x.jar`
2. 放入 `plugins` 目录
3. 启动服务器

### 第三步：验证安装

查看控制台日志：

```
[Emaki_Forge] Emaki Forge 正在启动...
[Emaki_Forge] 正在检查默认配置文件...
[Emaki_Forge] 已加载 X 个图纸
[Emaki_Forge] 已加载 X 个材料
[Emaki_Forge] 已加载 X 个配方
[Emaki_Forge] Emaki Forge 启动完成!
```

使用命令验证：

```
/jiuwuforge help
```

## 配置文件详解

### config.yml 主配置

```yaml
# Emaki Forge 主配置文件
# 版本: 1.4.0

# 配置文件版本号 (请勿修改)
config_version: "1.0.0"

# 语言设置
language: "zh_CN"

# 是否释放默认数据文件 (blueprints/materials/recipes)
# 设为 false 可避免覆盖自定义配置
release_default_data: true

# 品质系统配置
quality:
  tiers:
    - "劣质-60-0.9"
    - "普通-30-1.0"
    - "精良-10-1.1"
    - "卓越-5-1.2"
    - "传说-1-1.5"
  default_tier: "普通"
  guarantee:
    enabled: true
    threshold: 10
    minimum: "普通"
  item_meta:
    enabled: false
    tiers:
      普通:
        name_modifications: []
        lore_operations: []
      精良:
        name_modifications:
          - operation: "append_suffix"
            value: " <aqua>[精良]</aqua>"
        lore_operations:
          - operation: "append_lines"
            content:
              - "<aqua>品质: 精良</aqua>"

# 数值格式化配置
number_format:
  default: "0.##"
  integer: "0"
  percentage: "0.##%"

# 权限配置
permission:
  op_bypass: false

# 条件配置
condition:
  invalid_as_failure: true

# 历史记录配置
history:
  enabled: true
  auto_save: true
  save_interval: 6000

# 调试模式
debug: false
```

### 配置项说明

#### 基础配置

| 配置路径 | 类型 | 默认值 | 说明 |
|---------|------|--------|------|
| `config_version` | String | "1.0.0" | 配置文件版本 |
| `language` | String | "zh_CN" | 语言文件名 |
| `release_default_data` | Boolean | true | 是否释放默认数据文件 |

#### 品质系统配置

| 配置路径 | 类型 | 说明 |
|---------|------|------|
| `quality.tiers` | List | 品质等级列表，格式：名称-权重-倍率 |
| `quality.default_tier` | String | 默认品质 |
| `quality.guarantee.enabled` | Boolean | 是否启用保底机制 |
| `quality.guarantee.threshold` | Integer | 保底触发次数 |
| `quality.guarantee.minimum` | String | 保底最低品质 |
| `quality.item_meta.enabled` | Boolean | 是否启用品质元数据修改 |

#### 权限与条件

| 配置路径 | 类型 | 说明 |
|---------|------|------|
| `permission.op_bypass` | Boolean | OP是否绕过权限检查 |
| `condition.invalid_as_failure` | Boolean | 无效条件是否视为失败 |

#### 历史记录

| 配置路径 | 类型 | 说明 |
|---------|------|------|
| `history.enabled` | Boolean | 是否启用历史记录 |
| `history.auto_save` | Boolean | 是否自动保存 |
| `history.save_interval` | Integer | 保存间隔（tick） |

### 语言文件 (zh_CN.yml)

语言文件支持 MiniMessage 格式，可以自定义所有消息内容：

```yaml
# 通用消息
general:
  prefix: "<gray>[ <gradient:#F2C46D:#C9703D>Emaki Forge</gradient> ]</gray>"
  no_permission: "<red>你没有权限执行此操作!</red>"
  reload_success: "<green>配置重载成功!</green>"

# 锻造相关
forge:
  error:
    no_recipe: "<red>未找到配方!</red>"
    permission_denied: "<red>你没有权限使用此配方!</red>"
  success:
    crafted: "<green>锻造成功! 获得了 {item}</green>"
```

## 命令使用

### 基础命令

#### 打开锻造台

```
/jiuwuforge forge
或
/jiuwuforge
```

#### 打开配方图鉴

```
/jiuwuforge book
```

#### 重载配置

```
/jiuwuforge reload
```

重载后会显示加载统计：

```
已重载: 配置/语言/图纸(2)/材料(6)/配方(1)/GUI(2)
```

#### 列出配置项

```
/jiuwuforge list recipes    # 列出所有配方
/jiuwuforge list blueprints # 列出所有图纸
/jiuwuforge list materials  # 列出所有材料
```

#### 调试模式

```
/jiuwuforge debug
```

## 权限配置

### 在权限插件中配置

#### LuckPerms 示例

```
/lp user <玩家> permission set jiuwuforge.use true
/lp user <玩家> permission set jiuwuforge.book true
/lp group admin permission set jiuwuforge.admin true
```

#### 配方权限

每个配方可以配置独立的权限节点：

```yaml
# recipes/legendary_sword.yml
permission: "jiuwuforge.recipe.legendary_sword"
```

玩家需要拥有对应权限才能使用该配方：

```
/lp user <玩家> permission set jiuwuforge.recipe.legendary_sword true
```

## 常见问题

### Q: 提示 "Unknown dependency: Emaki_CoreLib"

**解决方案**：
1. 确认已安装 Emaki CoreLib
2. 确保 CoreLib 版本兼容
3. 检查 CoreLib 是否正常启动

### Q: 配置文件未生成

**解决方案**：
1. 检查 `plugins` 目录权限
2. 确认磁盘空间充足
3. 查看控制台是否有错误日志

### Q: 自定义物品无法识别

**解决方案**：
1. 确认 NeigeItems 或 CraftEngine 已安装
2. 检查物品 ID 格式是否正确
3. 验证物品是否存在于对应插件中

### Q: 如何禁用默认数据生成？

**解决方案**：

在 `config.yml` 中设置：

```yaml
release_default_data: false
```

## 卸载指南

### 完全卸载

1. 停止服务器
2. 删除 `plugins/Emaki_Forge.jar`
3. 删除 `plugins/Emaki_Forge/` 目录
4. 启动服务器

### 保留数据卸载

1. 停止服务器
2. 删除 `plugins/Emaki_Forge.jar`
3. 保留 `plugins/Emaki_Forge/data/` 目录
4. 重新安装时数据会自动恢复
