# Emaki Forge - 常见问题

## 安装与配置

### Q: 提示 "Unknown dependency: Emaki_CoreLib"

**原因**：未安装 Emaki CoreLib 或版本不兼容。

**解决方案**：
1. 确认已下载并安装 Emaki CoreLib
2. 确保 CoreLib 版本与 Forge 版本兼容
3. 检查控制台是否有 CoreLib 的启动日志
4. 尝试重新下载最新版本

---

### Q: 配置文件未自动生成

**原因**：权限问题或磁盘空间不足。

**解决方案**：
1. 检查服务器对 `plugins` 目录的写入权限
2. 确认磁盘空间充足
3. 查看控制台是否有相关错误日志
4. 尝试手动创建 `plugins/Emaki_Forge` 目录

---

### Q: 如何禁用默认数据文件的自动释放？

**解决方案**：

在 `config.yml` 中设置：

```yaml
release_default_data: false
```

这样插件启动时不会覆盖您的自定义配置文件。

---

### Q: 如何更改语言？

**解决方案**：

1. 修改 `config.yml` 中的 `language` 配置：

```yaml
language: "en_US"
```

2. 创建对应的语言文件 `plugins/Emaki_Forge/lang/en_US.yml`
3. 重载配置：`/jiuwuforge reload`

---

## 图纸与材料

### Q: 自定义物品无法识别

**原因**：物品源前缀错误或物品不存在。

**解决方案**：
1. 确认 NeigeItems 或 CraftEngine 已正确安装
2. 检查物品 ID 格式：
   - NeigeItems：`ni-物品ID`
   - CraftEngine：`ce-命名空间:物品ID`
   - 原版：`v-物品类型`
3. 验证物品在对应插件中是否存在
4. 检查控制台是否有物品加载错误

---

### Q: 图纸标签匹配不生效

**原因**：标签配置或选择器格式错误。

**解决方案**：
1. 确认图纸配置了正确的标签：

```yaml
# blueprints/weapon_sword.yml
tags:
  - "weapon"
  - "sword"
```

2. 确认配方选择器格式正确：

```yaml
# recipes/sword.yml
blueprint_requirements:
  - requirement_mode: "any_of"
    blueprint_options:
      - selector:
          kind: "tag"
          value: "weapon"
        count: 1
```

---

### Q: 材料属性未生效

**原因**：效果配置格式错误或属性名不被支持。

**解决方案**：
1. 检查 `effects` 配置格式：

```yaml
effects:
  - type: "stat_contribution"
    stats:
      physical_damage:
        type: "constant"
        value: 5
```

2. 确认属性名与目标物品兼容
3. 检查是否有语法错误（YAML 缩进）

---

## 配方与锻造

### Q: 配方无法匹配

**可能原因及解决方案**：

| 原因 | 解决方案 |
|------|---------|
| 图纸不满足 | 检查图纸需求和选择器配置 |
| 材料不足 | 确认必需材料的种类和数量 |
| 条件不满足 | 检查条件表达式是否正确 |
| 权限不足 | 确认玩家拥有配方权限 |
| 容量超限 | 检查可选材料是否超过容量上限 |

---

### Q: 锻造后物品消失

**原因**：产出物品配置错误或物品源不可用。

**解决方案**：
1. 检查 `result.output_item.item` 配置是否正确
2. 确认产出物品在对应插件中存在
3. 查看控制台是否有错误日志
4. 检查玩家背包是否有空位

---

### Q: 条件表达式不生效

**原因**：表达式语法错误或变量不可用。

**解决方案**：
1. 确认表达式语法正确：

```yaml
conditions:
  - "%player_level% >= 10"           # 正确
  - "player_level >= 10"             # 错误：缺少 %
  - "%player_level% >= 10 && %player_exp% >= 1000"  # 正确
```

2. 确认使用的变量已注册（PlaceholderAPI）
3. 检查 `condition_type` 配置是否正确

---

### Q: 如何设置多条件满足任意一个即可？

**解决方案**：

使用 `any_of` 条件类型：

```yaml
condition_type: "any_of"
conditions:
  - "%player_level% >= 50"
  - "permission == jiuwuforge.vip"
  - "%jiuwuforge_reputation% >= 1000"
```

---

### Q: 如何设置至少满足 N 个条件？

**解决方案**：

使用 `at_least` 条件类型：

```yaml
condition_type: "at_least"
conditions:
  - "%player_level% >= 30"
  - "%player_exp% >= 5000"
  - "permission == jiuwuforge.expert"
  - "%jiuwuforge_crafted_count% >= 10"
condition_required_count: 2  # 至少满足 2 个
```

## 品质系统

### Q: 品质保底不触发

**原因**：保底配置错误或未启用。

**解决方案**：
1. 确认保底已启用：

```yaml
quality:
  guarantee:
    enabled: true
```

2. 检查 `threshold` 值是否合理
3. 确认 `minimum` 品质存在于 `tiers` 中

---

### Q: 品质名称修改不显示

**原因**：元数据修改未启用或配置错误。

**解决方案**：
1. 确认 `item_meta.enabled` 为 `true`：

```yaml
quality:
  item_meta:
    enabled: true
    tiers:
      精良:
        name_modifications:
          - operation: "append_suffix"
            value: " <aqua>[精良]</aqua>"
```

2. 确认品质名称与 `tiers` 中定义的一致

---

### Q: 如何为特定配方设置不同的品质池？

**解决方案**：

在配方中使用 `custom_pool`：

```yaml
quality:
  enabled: true
  custom_pool:
    - "精良-40-1.1"
    - "卓越-35-1.2"
    - "传说-25-1.5"
```

## GUI 界面

### Q: GUI 无法打开

**原因**：模板文件不存在或配置错误。

**解决方案**：
1. 确认 GUI 模板文件存在：`plugins/Emaki_Forge/gui/forge_gui.yml`
2. 检查 YAML 语法是否正确
3. 确认 `id` 配置与引用一致
4. 查看控制台错误日志

---

### Q: 槽位点击无反应

**原因**：槽位未配置 action 或 action 无效。

**解决方案**：
1. 确认槽位配置了正确的 action：

```yaml
slots:
  confirm:
    slots: [40]
    action: "confirm"  # 必须配置
```

2. 确认 action 值是系统保留值之一
3. 检查槽位索引是否有效

---

### Q: GUI 变量不替换

**原因**：变量名错误或格式不正确。

**解决方案**：
1. 确认使用正确的变量格式：`{variable}`
2. 检查变量名是否正确：

```yaml
lore:
  - "<gray>当前容量: <yellow>{current}</yellow>"  # 正确
  - "<gray>当前容量: <yellow>$current</yellow>"   # 错误
```

---

## 性能与优化

### Q: 服务器卡顿

**可能原因及解决方案**：

| 原因 | 解决方案 |
|------|---------|
| 配方数量过多 | 合并相似配方，减少总数 |
| 条件计算复杂 | 简化条件表达式 |
| 玩家数据保存频繁 | 增大 `save_interval` |
| 自定义物品加载慢 | 优化 NeigeItems/CraftEngine 配置 |

---

### Q: 如何优化大量配方？

**解决方案**：
1. 使用标签匹配代替 ID 匹配
2. 减少条件数量
3. 避免复杂的表达式计算
4. 定期清理未使用的配方

---

### Q: 玩家数据保存影响性能

**解决方案**：

调整保存间隔：

```yaml
history:
  enabled: true
  auto_save: true
  save_interval: 12000  # 10分钟 (12000 ticks)
```

---

## 兼容性问题

### Q: 与其他锻造插件冲突

**原因**：命令或权限冲突。

**解决方案**：
1. 检查是否有命令冲突
2. 考虑禁用其他锻造插件
3. 使用不同的命令前缀

---

### Q: PlaceholderAPI 变量不可用

**原因**：PlaceholderAPI 未安装或变量未注册。

**解决方案**：
1. 确认 PlaceholderAPI 已安装
2. 安装所需的扩展：`/papi ecloud download Player`
3. 重载 PlaceholderAPI：`/papi reload`

---

### Q: 与特定版本 Minecraft 不兼容

**解决方案**：
1. 确认插件支持的 Minecraft 版本
2. 使用推荐的 Paper 版本
3. 查看更新日志了解版本支持情况

---

## 调试技巧

### 启用调试模式

```yaml
debug: true
```

启用后会在控制台输出详细的调试信息。

### 使用调试命令

```
/jiuwuforge debug
```

查看当前配置加载状态。

### 检查配方列表

```
/jiuwuforge list recipes
/jiuwuforge list blueprints
/jiuwuforge list materials
```

确认配置是否正确加载。

### 查看控制台日志

启动时检查是否有错误：

```
[Emaki_Forge] 已加载 X 个图纸
[Emaki_Forge] 已加载 X 个材料
[Emaki_Forge] 已加载 X 个配方
```
