# Emaki Forge - 配方系统

## 概述

配方（Recipe）是锻造系统的核心定义，规定了锻造所需的图纸、材料、条件以及产出物品。通过灵活的配方配置，可以创建各种独特的锻造配方。

---

## 配置文件位置

```
plugins/Emaki_Forge/recipes/
├── flame_sword.yml      # 烈焰之剑配方
└── custom_recipe.yml    # 自定义配方
```

## 基础配置结构

```yaml
id: "flame_sword"
display_name: "<gold>烈焰之剑锻造配方</gold>"

target_item:
  item: "ni-base_sword_iron"
  forge_capacity: 50

blueprint_requirements:
  - requirement_mode: "all_of"
    blueprint_options:
      - selector:
          kind: "id"
          value: "weapon_sword"
        count: 1

required_materials:
  - id: "iron_essence"
    count: 3
  - id: "flame_crystal"
    count: 1

optional_materials:
  enabled: true
  whitelist: []
  blacklist:
    - "cursed_stone"
  max_count: 5

condition_type: "all_of"
conditions:
  - "%player_level% >= 10"

quality:
  enabled: true
  guarantee:
    enabled: true
    attempts: 10
    minimum: "普通"

gui:
  template: "forge_gui"
  slots:
    blueprint_inputs: [10, 11]
    target_item: [13]
    required_materials: [19, 20, 21]
    optional_materials: [28, 29, 30, 31, 32]
    capacity_display: [22]
    confirm: [40]
    result_preview: [24]

result:
  output_item:
    item: "ni-flame_sword"
  success_message: "forge.success.crafted"
  sound: "minecraft:block.anvil.use-1-1"

permission: "jiuwuforge.recipe.flame_sword"
```

## 配置项详解

### id

**类型**：String  
**必填**：是  
**说明**：配方的唯一标识符

```yaml
id: "flame_sword"
```

---

### display_name

**类型**：String  
**必填**：否（默认使用 id）  
**说明**：配方的显示名称

```yaml
display_name: "<gold>烈焰之剑锻造配方</gold>"
```

---

### target_item

**类型**：配置节点  
**必填**：否  
**说明**：目标物品（锻造的基础物品）

```yaml
target_item:
  item: "ni-base_sword_iron"    # 物品源
  forge_capacity: 50            # 锻造容量（可在此设置）
```

**说明**：
- 如果不设置 `target_item`，配方将不需要目标物品
- `forge_capacity` 可以在此设置，也可以使用图纸的容量

---

### blueprint_requirements

**类型**：List  
**必填**：否  
**说明**：图纸需求配置

```yaml
blueprint_requirements:
  - requirement_mode: "all_of"
    blueprint_options:
      - selector:
          kind: "id"
          value: "weapon_sword"
        count: 1
  - requirement_mode: "any_of"
    blueprint_options:
      - selector:
          kind: "id"
          value: "universal_blueprint"
        count: 1
      - selector:
          kind: "tag"
          value: "weapon"
        count: 1
```

#### requirement_mode

| 模式 | 说明 |
|------|------|
| `all_of` | 必须满足所有选项 |
| `any_of` | 满足任一选项即可 |

#### selector

| 类型 | 说明 |
|------|------|
| `kind: "id"` | 通过图纸ID匹配 |
| `kind: "tag"` | 通过图纸标签匹配 |

---

### required_materials

**类型**：List  
**必填**：否  
**说明**：必需材料列表

```yaml
required_materials:
  - id: "iron_essence"
    count: 3
  - id: "flame_crystal"
    count: 1
```

---

### optional_materials

**类型**：配置节点  
**必填**：否  
**说明**：可选材料配置

```yaml
optional_materials:
  enabled: true              # 是否启用可选材料
  whitelist: []              # 白名单（空=允许所有）
  blacklist:                 # 黑名单
    - "cursed_stone"
  max_count: 5               # 最大数量
```

---

### conditions

**类型**：List\<String\>  
**必填**：否  
**说明**：锻造条件列表

```yaml
condition_type: "all_of"
conditions:
  - "%player_level% >= 10"
  - "%player_exp% >= 1000"
  - "permission == jiuwuforge.apprentice"
```

#### condition_type

| 模式 | 说明 |
|------|------|
| `all_of` | 所有条件都满足 |
| `any_of` | 任一条件满足 |
| `at_least` | 至少满足 N 个 |
| `exactly` | 恰好满足 N 个 |

---

### quality

**类型**：配置节点  
**必填**：否  
**说明**：品质配置

```yaml
quality:
  enabled: true
  custom_pool: []            # 自定义品质池
  guarantee:
    enabled: true            # 启用保底
    attempts: 10             # 保底次数
    minimum: "普通"          # 保底最低品质
```

---

### gui

**类型**：配置节点  
**必填**：否  
**说明**：GUI 配置覆盖

```yaml
gui:
  template: "forge_gui"      # 使用的GUI模板
  slots:
    blueprint_inputs: [10, 11]
    target_item: [13]
    required_materials: [19, 20, 21]
    optional_materials: [28, 29, 30, 31, 32]
    capacity_display: [22]
    confirm: [40]
    result_preview: [24]
```

---

### result

**类型**：配置节点  
**必填**：否  
**说明**：锻造结果配置

```yaml
result:
  output_item:
    item: "ni-flame_sword"
  success_message: "forge.success.crafted"
  sound: "minecraft:block.anvil.use-1-1"
  name_modifications:        # 名称修改
    - operation: "append_suffix"
      value: " <red>[烈焰]</red>"
  lore_operations:           # 描述操作
    - operation: "append_lines"
      content:
        - "<red>火焰伤害: +{fire_damage}"
```

---

### permission

**类型**：String  
**必填**：否  
**说明**：使用此配方所需的权限

```yaml
permission: "jiuwuforge.recipe.flame_sword"
```

## 完整配置示例

### 示例1：基础武器配方

```yaml
id: "iron_sword"
display_name: "<white>铁剑锻造配方</white>"

target_item:
  item: "ni-base_sword_iron"
  forge_capacity: 30

blueprint_requirements:
  - requirement_mode: "any_of"
    blueprint_options:
      - selector:
          kind: "id"
          value: "weapon_sword"
        count: 1
      - selector:
          kind: "id"
          value: "universal_blueprint"
        count: 1

required_materials:
  - id: "iron_essence"
    count: 2

optional_materials:
  enabled: true
  whitelist: []
  blacklist: []
  max_count: 3

condition_type: "all_of"
conditions:
  - "%player_level% >= 5"

quality:
  enabled: true
  guarantee:
    enabled: false

gui:
  template: "forge_gui"

result:
  output_item:
    item: "ni-iron_sword"
  success_message: "forge.success.crafted"
  sound: "minecraft:block.anvil.use-1-1"

permission: "jiuwuforge.recipe.iron_sword"
```

### 示例4：多图纸配方

```yaml

  item: "ni-base_armor_chain"
  forge_capacity: 80

      - selector:

  - id: "iron_essence"

  - id: "flame_crystal"
```

## 配方匹配流程

### 1. 目标物品检查

如果配方需要目标物品，检查玩家放入的物品是否匹配。

### 2. 图纸检查

验证玩家放入的图纸是否满足 `blueprint_requirements`。

### 3. 必需材料检查

验证必需材料的种类和数量。

### 4. 可选材料检查

验证可选材料是否在白名单/黑名单范围内。

### 5. 容量检查

计算可选材料的总容量消耗，确保不超过上限。

### 6. 条件检查

评估所有条件是否满足。

### 7. 权限检查

验证玩家是否有使用此配方的权限。

## 最佳实践

### 1. 合理设置条件难度

```yaml
# 简单配方
conditions:
  - "%player_level% >= 5"

# 中等配方
conditions:
  - "%player_level% >= 20"
  - "%jiuwuforge_reputation% >= 500"

# 困难配方
conditions:
  - "%player_level% >= 50"
  - "%jiuwuforge_reputation% >= 3000"
  - "permission == jiuwuforge.master"
```

### 2. 使用标签简化图纸匹配

```yaml
# 推荐：使用标签
blueprint_requirements:
  - requirement_mode: "any_of"
    blueprint_options:
      - selector:
          kind: "tag"
          value: "weapon"
        count: 1

# 不推荐：列出所有ID
blueprint_requirements:
  - requirement_mode: "any_of"
    blueprint_options:
      - selector:
          kind: "id"
          value: "weapon_sword"
        count: 1
      - selector:
          kind: "id"
          value: "weapon_axe"
        count: 1
      # ... 更多
```

### 3. 提供清晰的错误提示

通过语言文件配置错误消息：

```yaml
forge:
  error:
    no_target_item: "<red>请放入目标物品!</red>"
    invalid_target_item: "<red>目标物品不匹配!</red>"
    blueprint_required: "<red>缺少必要的图纸!</red>"
    material_required: "<red>缺少必要的材料!</red>"
    capacity_exceeded: "<red>锻造容量超限! 当前: {current}/{max}</red>"
```

## 故障排查

### 配方无法匹配

**检查项**：
1. 图纸、材料是否正确放入
2. 条件是否满足
3. 权限是否拥有
4. 容量是否超限

### 锻造失败

**检查项**：
1. 查看控制台错误日志
2. 检查 `output_item` 是否存在
3. 验证品质配置是否正确

### GUI 显示异常

**检查项**：
1. GUI 模板是否存在
2. 槽位配置是否正确
3. 槽位索引是否有效
