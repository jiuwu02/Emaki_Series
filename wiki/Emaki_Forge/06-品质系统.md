# Emaki Forge - 品质系统

## 概述

品质系统为锻造产物提供随机品质属性，影响产物的属性倍率和显示效果。通过品质系统，可以为玩家带来惊喜和期待感，同时保底机制确保长期体验的公平性。

---

## 品质配置

### 主配置 (config.yml)

```yaml
quality:
  # 品质等级列表
  # 格式: 名称-权重-倍率
  tiers:
    - "劣质-60-0.9"
    - "普通-30-1.0"
    - "精良-10-1.1"
    - "卓越-5-1.2"
    - "传说-1-1.5"
  
  # 默认品质
  default_tier: "普通"
  
  # 保底机制
  guarantee:
    enabled: true
    threshold: 10
    minimum: "普通"
  
  # 品质元数据修改
  item_meta:
    enabled: false
    tiers:
      普通:
        name_modifications: []
        lore_operations: []
      精良:
        name_modifications:
          - operation: "append_suffix"
            value: " <aqua>[精良]</aqua>"
        lore_operations:
          - operation: "append_lines"
            content:
              - "<aqua>品质: 精良</aqua>"
```

## 品质等级配置

### 格式说明

```
名称-权重-倍率
```

| 字段 | 说明 |
|------|------|
| 名称 | 品质的显示名称 |
| 权重 | 随机抽取的权重值（越大越容易被抽中） |
| 倍率 | 属性倍率（影响最终属性值） |

### 权重计算

品质抽取采用加权随机算法：

```
概率 = 品质权重 / 所有权重之和
```

**示例计算**：

```yaml
tiers:
  - "劣质-60-0.9"    # 概率: 60/(60+30+10+5+1) = 60/106 ≈ 56.6%
  - "普通-30-1.0"    # 概率: 30/106 ≈ 28.3%
  - "精良-10-1.1"    # 概率: 10/106 ≈ 9.4%
  - "卓越-5-1.2"     # 概率: 5/106 ≈ 4.7%
  - "传说-1-1.5"     # 概率: 1/106 ≈ 0.9%
```

## 保底机制

### 配置说明

```yaml
guarantee:
  enabled: true       # 是否启用保底
  threshold: 10       # 保底触发次数
  minimum: "普通"     # 保底最低品质
```

### 工作原理

1. 系统记录玩家连续获得低于最低品质的次数
2. 当次数达到 `threshold` 时，强制获得至少 `minimum` 品质
3. 获得保底品质后，计数器重置

### 示例

```yaml
guarantee:
  enabled: true
  threshold: 10
  minimum: "普通"
```

- 玩家连续 10 次获得"劣质"品质
- 第 11 次锻造必定获得至少"普通"品质
- 保底触发后计数器归零

## 品质元数据修改

### 名称修改

```yaml
item_meta:
  enabled: true
  tiers:
    精良:
      name_modifications:
        - operation: "append_suffix"
          value: " <aqua>[精良]</aqua>"
```

**操作类型**：

| 操作 | 说明 |
|------|------|
| `append_suffix` | 添加后缀 |
| `prepend_prefix` | 添加前缀 |
| `replace` | 替换名称 |

### 描述修改

```yaml
item_meta:
  enabled: true
  tiers:
    传说:
      lore_operations:
        - operation: "prepend_lines"
          content:
            - "<gold>══════ 传说品质 ══════"
        - operation: "append_lines"
          content:
            - "<gold>此物品蕴含传说之力"
```

**操作类型**：

| 操作 | 说明 |
|------|------|
| `append_lines` | 追加行 |
| `prepend_lines` | 插入行到开头 |
| `insert_below` | 插入到匹配行下方 |
| `insert_above` | 插入到匹配行上方 |

## 配方级品质配置

### 自定义品质池

配方可以覆盖全局品质配置：

```yaml
quality:
  enabled: true
  custom_pool:
    - "精良-40-1.1"
    - "卓越-35-1.2"
    - "传说-25-1.5"
```

### 配方级保底

```yaml
quality:
  enabled: true
  guarantee:
    enabled: true
    attempts: 5
    minimum: "精良"
```

## 完整配置示例

### 示例1：标准品质系统

```yaml
quality:
  tiers:
    - "劣质-50-0.8"
    - "普通-35-1.0"
    - "精良-10-1.15"
    - "卓越-4-1.3"
    - "传说-1-1.6"
  
  default_tier: "普通"
  
  guarantee:
    enabled: true
    threshold: 15
    minimum: "普通"
  
  item_meta:
    enabled: true
    tiers:
      劣质:
        name_modifications:
          - operation: "append_suffix"
            value: " <dark_gray>[劣质]</dark_gray>"
        lore_operations:
          - operation: "append_lines"
            content:
              - "<dark_gray>品质: 劣质 (属性-20%)"
      
      普通:
        name_modifications: []
        lore_operations: []
      
      精良:
        name_modifications:
          - operation: "append_suffix"
            value: " <aqua>[精良]</aqua>"
        lore_operations:
          - operation: "append_lines"
            content:
              - "<aqua>品质: 精良 (属性+15%)"
      
      卓越:
        name_modifications:
          - operation: "append_suffix"
            value: " <gold>[卓越]</gold>"
        lore_operations:
          - operation: "append_lines"
            content:
              - "<gold>品质: 卓越 (属性+30%)"
      
      传说:
        name_modifications:
          - operation: "prepend_prefix"
            value: "<gold>传说之</gold> "
          - operation: "append_suffix"
            value: " <red>[传说]</red>"
        lore_operations:
          - operation: "prepend_lines"
            content:
              - "<gold>══════ 传说品质 ══════"
          - operation: "append_lines"
            content:
              - "<gold>品质: 传说 (属性+60%)"
              - "<yellow>此物品蕴含传说之力"
```

## 属性倍率应用

### 倍率计算

品质倍率应用于所有属性贡献：

```
最终属性 = 基础属性 × 品质倍率
```

### 示例

```yaml
# 材料贡献
physical_damage: 10
max_health: 5

# 品质倍率: 1.2 (卓越)
# 最终属性
physical_damage: 10 × 1.2 = 12
max_health: 5 × 1.2 = 6
```

## 品质显示

### 在锻造结果中显示

```yaml
result:
  output_item:
    item: "ni-flame_sword"
  success_message: "forge.success.crafted"

# 语言文件
forge:
  success:
    crafted: "<green>锻造成功! 获得了 {item}</green>"
    quality: "<gold>品质: {quality} (倍率: {multiplier}x)</gold>"
```

### 在配方图鉴中显示

配方图鉴会显示玩家是否已完成锻造，但不显示品质信息。

## 最佳实践

### 1. 平衡权重分布

```yaml
# 推荐：金字塔分布
tiers:
  - "劣质-50-0.9"    # 底层，概率最高
  - "普通-30-1.0"
  - "精良-15-1.1"
  - "卓越-4-1.2"
  - "传说-1-1.5"     # 顶层，概率最低
```

### 2. 合理设置保底

```yaml
# 普通：15次保底
guarantee:
  threshold: 15
  minimum: "普通"

# 高级配方：5次保底
guarantee:
  threshold: 5
  minimum: "精良"
```

### 3. 品质名称一致性

确保品质名称在所有配置中一致：

```yaml
# 正确
tiers:
  - "精良-10-1.1"
guarantee:
  minimum: "精良"    # 与 tiers 中的名称一致

# 错误
tiers:
  - "精良-10-1.1"
guarantee:
  minimum: "优秀"    # 不存在于 tiers 中
```

## 故障排查

### 品质未生效

**检查项**：
1. 配方中 `quality.enabled` 是否为 `true`
2. `tiers` 配置格式是否正确
3. 品质名称是否一致

### 保底不触发

**检查项**：
1. `guarantee.enabled` 是否为 `true`
2. `threshold` 值是否合理
3. `minimum` 品质是否存在于 `tiers` 中

### 元数据修改失败

**检查项**：
1. `item_meta.enabled` 是否为 `true`
2. 品质名称是否正确
3. 操作格式是否正确
