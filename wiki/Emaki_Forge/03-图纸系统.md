# Emaki Forge - 图纸系统

## 概述

图纸（Blueprint）是锻造系统的核心组件之一，定义了锻造所需的图纸类型和属性。图纸可以理解为锻造的"模板"，决定了玩家能够锻造什么类型的物品。

---

## 图纸配置

### 配置文件位置

```
plugins/Emaki_Forge/blueprints/
├── universal_blueprint.yml    # 通用图纸
├── weapon_sword.yml           # 武器图纸
└── custom_blueprint.yml       # 自定义图纸
```

### 基础配置结构

```yaml
id: "weapon_sword"
display_name: "<gold>武器锻造图纸</gold>"
description:
  - "<gray>用于锻造武器类装备"
  - "<gray>包含剑、刀等近战武器配方"

item: "ni-blueprint_weapon_sword"

tags:
  - "weapon"
  - "melee"
  - "sword"
```

---

## 配置项详解

### id

**类型**：String  
**必填**：是  
**说明**：图纸的唯一标识符，用于配方中引用

```yaml
id: "weapon_sword"
```

---

### display_name

**类型**：String  
**必填**：否（默认使用 id）  
**说明**：图纸的显示名称，支持 MiniMessage 格式

```yaml
display_name: "<gold>武器锻造图纸</gold>"
```

---

### description

**类型**：List\<String\>  
**必填**：否  
**说明**：图纸的描述文本，支持 MiniMessage 格式

```yaml
description:
  - "<gray>用于锻造武器类装备"
  - "<gray>包含剑、刀等近战武器配方"
  - "<yellow>需要锻造等级 10 以上"
```

---

### item

**类型**：String（物品源简写）  
**必填**：是  
**说明**：图纸对应的物品，支持多种物品源

```yaml
# NeigeItems 物品
item: "ni-blueprint_weapon_sword"

# CraftEngine 物品
item: "ce-namespace:blueprint_sword"

# 原版物品
item: "v-PAPER"
```

---

### tags

**类型**：List\<String\>  
**必填**：否  
**说明**：图纸的标签，用于配方中的标签匹配

```yaml
tags:
  - "weapon"
  - "melee"
  - "sword"
```

---

### forge_capacity

**类型**：Integer  
**必填**：否  
**说明**：图纸提供的锻造容量上限

```yaml
forge_capacity: 50
```

## 图纸匹配机制

### ID 匹配

通过图纸 ID 精确匹配：

```yaml
# 配方中的图纸要求
blueprint_requirements:
  - requirement_mode: "all_of"
    blueprint_options:
      - selector:
          kind: "id"
          value: "weapon_sword"
        count: 1
```

### 标签匹配

通过图纸标签模糊匹配：

```yaml
blueprint_requirements:
  - requirement_mode: "any_of"
    blueprint_options:
      - selector:
          kind: "tag"
          value: "weapon"
        count: 1
```

---

## 图纸需求模式

### all_of 模式

必须满足所有选项：

```yaml
blueprint_requirements:
  - requirement_mode: "all_of"
    blueprint_options:
      - selector:
          kind: "id"
          value: "weapon_sword"
        count: 1
      - selector:
          kind: "id"
          value: "universal_blueprint"
        count: 1
```

**说明**：需要同时放入武器图纸和通用图纸

---

### any_of 模式

满足任一选项即可：

```yaml
blueprint_requirements:
  - requirement_mode: "any_of"
    blueprint_options:
      - selector:
          kind: "id"
          value: "weapon_sword"
        count: 1
      - selector:
          kind: "tag"
          value: "weapon"
        count: 1
```

**说明**：放入武器图纸或任何带有 weapon 标签的图纸

## 锻造容量系统

### 容量概念

锻造容量限制了可选材料的使用量：

- 每个材料都有 `capacity_cost`（容量消耗）
- 图纸提供 `forge_capacity`（容量上限）
- 可选材料的总消耗不能超过图纸容量

### 容量计算

```
总消耗 = Σ(材料容量消耗 × 材料数量)
```

如果总消耗超过图纸容量，锻造将无法进行。

### 示例

```yaml
# 图纸配置
forge_capacity: 50

# 材料配置
materials:
  iron_essence:
    capacity_cost: 5
  flame_crystal:
    capacity_cost: 10
  rare_gem:
    capacity_cost: 20
```

玩家可以使用的组合：
- 10个铁精粹（10×5=50）✓
- 5个火焰结晶（5×10=50）✓
- 2个稀有宝石 + 2个火焰结晶（2×20+2×10=60）✗

## 最佳实践

### 1. 合理设置标签

使用层级化的标签体系：

```yaml
tags:
  - "weapon"        # 一级分类
  - "melee"         # 二级分类
  - "sword"         # 三级分类
```

### 2. 平衡锻造容量

根据图纸稀有度设置容量：

```yaml
# 普通图纸
forge_capacity: 30

# 稀有图纸
forge_capacity: 50

# 传说图纸
forge_capacity: 80
```

### 3. 提供清晰的描述

帮助玩家理解图纸用途：

```yaml
description:
  - "<gray>用于锻造武器类装备"
  - "<gray>锻造容量: <yellow>50</yellow>"
  - "<gray>可锻造: <white>剑、刀、斧"
  - ""
  - "<yellow>右键查看可锻造配方"
```

## 故障排查

### 图纸无法识别

**检查项**：
1. `item` 配置的物品是否存在
2. 物品源前缀是否正确
3. 对应插件（NeigeItems/CraftEngine）是否安装

### 图纸匹配失败

**检查项**：
1. 配方中的 selector 格式是否正确
2. 图纸 ID 或标签是否匹配
3. 图纸数量是否满足要求

### 容量计算异常

**检查项**：
1. 材料的 `capacity_cost` 是否配置
2. 图纸的 `forge_capacity` 是否设置
3. 数值是否为有效整数
