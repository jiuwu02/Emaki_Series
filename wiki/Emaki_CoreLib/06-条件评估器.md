# Emaki CoreLib - 条件评估器

## 概述

条件评估器（ConditionEvaluator）提供了一套灵活的条件判断系统，支持多种比较运算符、逻辑组合模式和占位符替换。该模块常用于配方解锁判断、权限验证、触发条件检测等场景。

---

## 核心概念

### 条件表达式格式

```
左值 运算符 右值
```

示例：
- `%player_level% >= 10`
- `{player_money} == 1000`
- `permission != null`

### 支持的运算符

| 运算符 | 说明 | 适用类型 |
|--------|------|---------|
| `==` | 等于 | 数值、字符串 |
| `!=` | 不等于 | 数值、字符串 |
| `<` | 小于 | 数值 |
| `<=` | 小于等于 | 数值 |
| `>` | 大于 | 数值 |
| `>=` | 大于等于 | 数值 |

---

## 基础用法

### 解析单个条件

```java
ParsedCondition parsed = ConditionEvaluator.parse("%player_level% >= 10");
// parsed.left() = "%player_level%"
// parsed.operator() = ">="
// parsed.right() = "10"
```

### 评估单个条件

```java
Boolean result = ConditionEvaluator.evaluateSingle(
    "%player_level% >= 10",
    text -> PlaceholderAPI.setPlaceholders(player, text)
);
// 返回 true 或 false
```

## 条件组合模式

### evaluate() 方法

```java
public static boolean evaluate(
    List<String> conditions,           // 条件列表
    String conditionType,              // 组合模式
    Integer requiredCount,             // 所需数量
    Function<String, String> placeholderReplacer,  // 占位符替换器
    boolean invalidAsFailure           // 无效条件处理方式
)
```

### 组合模式类型

| 模式 | 说明 | 示例 |
|------|------|------|
| `all_of` | 所有条件都满足 | 默认模式 |
| `any_of` | 任一条件满足 | - |
| `at_least` | 至少满足 N 个 | 需指定 requiredCount |
| `exactly` | 恰好满足 N 个 | 需指定 requiredCount |

## 使用示例

### 示例1：等级与权限检查

```yaml
# recipe.yml
conditions:
  - "%player_level% >= 30"
  - "%player_exp% >= 1000"
  - "permission == jiuwuforge.master"
condition_type: "all_of"
```

```java
public boolean canCraft(Player player, ConfigurationSection config) {
    List<String> conditions = Texts.asStringList(config.get("conditions"));
    String type = config.getString("condition_type", "all_of");
    
    return ConditionEvaluator.evaluate(
        conditions,
        type,
        null,
        text -> PlaceholderAPI.setPlaceholders(player, text),
        true  // 无效条件视为失败
    );
}
```

## 占位符替换

### 使用 PlaceholderAPI

```java
Function<String, String> replacer = text -> 
    PlaceholderAPI.setPlaceholders(player, text);

boolean result = ConditionEvaluator.evaluateSingle(
    "%player_level% >= 10",
    replacer
);
```

### 自定义替换逻辑

```java
Map<String, Object> variables = Map.of(
    "player_level", player.getLevel(),
    "player_money", economy.getBalance(player)
);

Function<String, String> replacer = text -> {
    for (Map.Entry<String, Object> entry : variables.entrySet()) {
        text = text.replace("{" + entry.getKey() + "}", String.valueOf(entry.getValue()));
    }
    return text;
};
```

### 字符串相等

```yaml

  - "{player_rank} == VIP"
  - "{guild_name} == DragonKnights"
```

### 字符串不等

```yaml

  - "{player_status} != banned"
  - "{account_type} != guest"
```

右值可以使用引号包裹，避免空格问题：

```yaml

  - "{player_title} == \"Master Crafter\""
  - "{guild_tag} == 'DK'"
```

## 无效条件处理

### invalidAsFailure 参数

| 值 | 行为 |
|----|------|
| `true` | 无效条件视为失败，可能导致整体失败 |
| `false` | 无效条件被忽略，继续评估其他条件 |

### 示例

```java
List<String> conditions = List.of(
    "%player_level% >= 10",
    "invalid condition",  // 无法解析
    "%player_exp% >= 100"
);

// invalidAsFailure = true
// 由于存在无效条件，整体可能失败

// invalidAsFailure = false
// 无效条件被忽略，只评估有效条件
```

## 完整示例

### 配方解锁系统

```java
public class RecipeUnlockManager {
    
    public boolean checkUnlock(Player player, Recipe recipe) {
        if (recipe.conditions().isEmpty()) {
            return true;
        }
        
        return ConditionEvaluator.evaluate(
            recipe.conditions(),
            recipe.conditionType(),
            recipe.conditionRequiredCount(),
            this::replacePlaceholders,
            plugin.appConfig().invalidAsFailure()
        );
    }
    
    private String replacePlaceholders(String text) {
        // 先处理 PlaceholderAPI
        text = PlaceholderAPI.setPlaceholders(player, text);
        
        // 再处理自定义变量
        text = text.replace("{player_level}", String.valueOf(player.getLevel()));
        text = text.replace("{player_world}", player.getWorld().getName());
        
        return text;
    }
}
```

### 配置示例


### 动态条件生成

```java
public List<String> generateConditions(Player player, String recipeId) {
    List<String> conditions = new ArrayList<>();
    
    // 基于玩家等级动态生成
    int requiredLevel = getRequiredLevel(recipeId);
    conditions.add("%player_level% >= " + requiredLevel);
    
    // 基于配方类型添加条件
    RecipeType type = getRecipeType(recipeId);
    switch (type) {
        case WEAPON -> conditions.add("%player_combat_skill% >= 50");
        case ARMOR -> conditions.add("%player_defense_skill% >= 50");
        case ACCESSORY -> conditions.add("%player_magic_skill% >= 50");
    }
    
    return conditions;
}
```

## 最佳实践

### 1. 条件可读性

使用清晰的变量名和注释：

```yaml
conditions:
  # 玩家等级至少30级
  - "%player_level% >= 30"
  
  # 拥有足够的金币
  - "%player_balance% >= 1000"
  
  # 完成前置任务
  - "%quest_completed_tutorial% == true"
```

### 2. 错误处理

记录无效条件以便调试：

```java
public boolean evaluateWithLogging(List<String> conditions, Player player) {
    for (String condition : conditions) {
        ParsedCondition parsed = ConditionEvaluator.parse(condition);
        if (parsed == null) {
            plugin.getLogger().warning("无效的条件表达式: " + condition);
            continue;
        }
        
        Boolean result = ConditionEvaluator.evaluateSingle(
            condition,
            text -> replacePlaceholders(player, text)
        );
        
        if (plugin.appConfig().debug()) {
            plugin.getLogger().info("条件 '" + condition + "' 结果: " + result);
        }
    }
    
    return ConditionEvaluator.evaluate(
        conditions, "all_of", null,
        text -> replacePlaceholders(player, text),
        true
    );
}
```

### 3. 性能优化

缓存频繁使用的条件解析结果：

```java
private final Map<String, ParsedCondition> parsedCache = new ConcurrentHashMap<>();

public boolean evaluateCached(String condition, Function<String, String> replacer) {
    ParsedCondition parsed = parsedCache.computeIfAbsent(
        condition,
        ConditionEvaluator::parse
    );
    
    if (parsed == null) return false;
    
    return ConditionEvaluator.evaluateSingle(condition, replacer);
}
```
