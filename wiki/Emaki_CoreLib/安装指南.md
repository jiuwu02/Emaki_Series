# Emaki CoreLib - 安装指南

## 系统要求

### 必需环境

| 要求项 | 最低版本 | 推荐版本 |
|--------|---------|---------|
| Minecraft Server | 1.21 | 1.21+ |
| 服务端类型 | Paper | Paper (最新版) |
| Java Runtime | 21 | 21+ |
| 内存分配 | 2GB | 4GB+ |

### 兼容性说明

- **支持的服务端**：Paper、Purpur、Pufferfish 等 Paper 分支
- **不支持的服务端**：Spigot、CraftBukkit、Forge、Fabric

---

## 安装步骤

### 第一步：下载插件

将 `Emaki_CoreLib-x.x.x.jar` 文件放入服务器的 `plugins` 目录：

```
server/
├── plugins/
│   └── Emaki_CoreLib-1.0.0.jar    ← 放置在这里
├── server.jar
└── ...
```

### 第二步：启动服务器

首次启动时，插件会自动创建必要的目录结构和默认配置文件：

```
plugins/
└── Emaki_CoreLib/
    ├── config.yml
    └── lang/
        └── zh_CN.yml
```

### 第三步：验证安装

查看控制台日志，确认以下输出：

```
[Emaki_CoreLib] Emaki_CoreLib enabled.
```

使用 `/plugins` 命令确认插件状态为绿色（已启用）。

## 配置文件详解

### config.yml 主配置

```yaml
# Emaki CoreLib 默认配置
config_version: "1.0.0"

text:
  # 默认消息前缀
  default_prefix: "<gray>[<gold>Emaki CoreLib</gold>]</gray>"

item_source:
  # 是否允许自定义物品源注册表
  allow_custom_registry: true

gui:
  # GUI 默认点击音效
  default_sound:
    sound: "ui.button.click"
    volume: 1.0
    pitch: 1.0
```

### 配置项说明

| 配置路径 | 类型 | 默认值 | 说明 |
|---------|------|--------|------|
| `config_version` | String | "1.0.0" | 配置文件版本号，请勿修改 |
| `text.default_prefix` | String | 见上方 | 默认消息前缀，支持 MiniMessage 格式 |
| `item_source.allow_custom_registry` | Boolean | true | 是否允许插件注册自定义物品源解析器 |
| `gui.default_sound.sound` | String | "ui.button.click" | 默认点击音效名称 |
| `gui.default_sound.volume` | Double | 1.0 | 音效音量 (0.0-1.0) |
| `gui.default_sound.pitch` | Double | 1.0 | 音效音调 (0.5-2.0) |

## 作为开发依赖使用

### Maven 配置

在项目的 `pom.xml` 中添加：

```xml
<dependencies>
    <!-- Emaki CoreLib -->
    <dependency>
        <groupId>emaki.jiuwu.craft</groupId>
        <artifactId>emaki-corelib</artifactId>
        <version>1.0.0</version>
        <scope>provided</scope>
    </dependency>
</dependencies>
```

### plugin.yml 配置

在插件的 `plugin.yml` 中声明依赖：

```yaml
name: MyPlugin
main: com.example.MyPlugin
version: 1.0.0
api-version: '1.21'
depend:
  - Emaki_CoreLib    # 必需依赖
```

如果 CoreLib 是可选功能：

```yaml
softdepend:
  - Emaki_CoreLib    # 可选依赖
```

## 常见问题

### Q: 插件无法启动，提示 "UnsupportedClassVersionError"

**原因**：Java 版本过低

**解决方案**：升级到 Java 21 或更高版本

### Q: 提示 "Unknown dependency: Emaki_CoreLib"

**原因**：CoreLib 未正确安装或未在 `depend` 中声明

**解决方案**：
1. 确认 `Emaki_CoreLib.jar` 在 `plugins` 目录中
2. 检查依赖插件的 `plugin.yml` 是否正确声明依赖

### Q: 配置文件未生成

**原因**：可能是权限问题或磁盘空间不足

**解决方案**：
1. 检查服务器对 `plugins` 目录的写入权限
2. 确认磁盘空间充足
3. 尝试手动创建 `Emaki_CoreLib` 目录

### Q: 与其他插件冲突

**原因**：可能存在类路径冲突

**解决方案**：
1. 确保没有重复加载 CoreLib
2. 检查其他插件是否打包了相同依赖

## 卸载指南

### 完全卸载

1. 停止服务器
2. 删除 `plugins/Emaki_CoreLib.jar`
3. 删除 `plugins/Emaki_CoreLib/` 目录
4. 启动服务器

### 注意事项

- 卸载 CoreLib 前请确保没有其他插件依赖它
- 依赖 CoreLib 的插件将无法正常工作
