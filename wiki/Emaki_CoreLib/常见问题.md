# Emaki CoreLib - 常见问题

## 安装与配置

### Q1: 插件无法启动，提示 "UnsupportedClassVersionError"

**原因**：Java 版本过低

**解决方案**：
1. 检查当前 Java 版本：`java -version`
2. 升级到 Java 21 或更高版本
3. 在启动脚本中指定正确的 Java 路径

```bash
java -version
# 确保输出显示 21 或更高版本
```

---

### Q2: 提示 "Unknown dependency: Emaki_CoreLib"

**原因**：CoreLib 未正确安装或未声明依赖

**解决方案**：
1. 确认 `Emaki_CoreLib.jar` 在 `plugins` 目录中
2. 检查依赖插件的 `plugin.yml`：

```yaml
depend:
  - Emaki_CoreLib
```

---

### Q3: 配置文件未生成

**原因**：权限问题或磁盘空间不足

**解决方案**：
1. 检查服务器对 `plugins` 目录的写入权限
2. 确认磁盘空间充足
3. 手动创建目录：

```bash
mkdir plugins/Emaki_CoreLib
mkdir plugins/Emaki_CoreLib/lang
```

---

### Q4: 如何更改语言？

**解决方案**：
1. 编辑 `config.yml` 中的语言设置
2. 创建对应的语言文件（如 `en_US.yml`）
3. 重载插件或重启服务器

---

## GUI 系统

### Q5: GUI 打开后立即关闭

**原因**：可能存在其他插件冲突或事件处理异常

**解决方案**：
1. 检查控制台是否有错误日志
2. 确认没有其他插件拦截 InventoryOpenEvent
3. 验证 GuiTemplate 配置是否正确

---

### Q6: 点击 GUI 没有反应

**原因**：事件处理器未正确注册或 action 未设置

**解决方案**：
1. 确认 GuiService 已注册为监听器：

```java
Bukkit.getPluginManager().registerEvents(guiService, plugin);
```

2. 检查槽位的 action 配置：

```yaml
slots:
  button:
    slots: [13]
    action: "confirm"  # 确保设置了 action
```

---

### Q7: GUI 标题变量不替换

**原因**：变量格式错误或未传入替换值

**解决方案**：
1. 确保标题使用 `{variable}` 格式：

```yaml
title: "<dark_gray>商店 - {player}</dark_gray>"
```

2. 传入替换值：

```java
guiService.open(new GuiOpenRequest(
    plugin, player, template,
    Map.of("player", player.getName()),  // 替换值
    itemFactory, renderer, handler
));
```

---

### Q8: 如何自定义 GUI 点击音效？

**解决方案**：

在槽位配置中添加音效：

```yaml
slots:
  button:
    slots: [13]
    item: "DIAMOND"
    click:
      sound: "ui.button.click"
      volume: 1.0
      pitch: 1.0
```

或使用简写格式：

```yaml
click: "ui.button.click-1.0-1.0"
```

## 物品源系统

### Q9: NeigeItems 物品无法识别

**原因**：NeigeItems 未安装或 API 桥接失败

**解决方案**：
1. 确认 NeigeItems 已安装并启用
2. 检查控制台是否有桥接失败的警告
3. 验证物品 ID 格式：

```yaml
item: "ni-flame_sword"  # 正确格式
```

---

### Q10: CraftEngine 物品创建失败

**原因**：命名空间格式错误或物品不存在

**解决方案**：
1. 使用正确的命名空间格式：

```yaml
item: "ce-namespace:item_id"
```

2. 确认物品在 CraftEngine 中存在
3. 检查 CraftEngine 是否正确加载

---

### Q11: 如何添加自定义物品源类型？

**解决方案**：

注册自定义解析器：

```java
ItemSourceRegistry registry = ItemSourceRegistry.system();

registry.registerParser(shorthand -> {
    if (shorthand.startsWith("myplugin-")) {
        String id = shorthand.substring("myplugin-".length());
        return new ItemSource(MyCustomType.INSTANCE, id);
    }
    return null;
});
```

## 表达式引擎

### Q12: 表达式计算结果为 0

**原因**：表达式无效或变量未定义

**解决方案**：
1. 检查表达式语法：

```java
// 正确
"{level} * 10 + {base}"

// 错误（缺少花括号）
"level * 10 + base"
```

2. 确保所有变量都已传入：

```java
Map<String, Object> variables = Map.of(
    "level", player.getLevel(),
    "base", 100
);
double result = ExpressionEngine.evaluate(expression, variables);
```

---

### Q13: 随机分布生成的值超出范围

**原因**：高斯分布可能产生超出限制的值

**解决方案**：
设置 min 和 max 参数：

```yaml
value:
  type: "gaussian"
  mean: 100
  std_dev: 15
  min: 50   # 最小值限制
  max: 150  # 最大值限制
```

---

### Q14: 如何实现复杂的随机逻辑？

**解决方案**：

使用嵌套配置：

```yaml
complex_value:
  type: "uniform"
  min:
    type: "gaussian"
    mean: 10
    std_dev: 2
  max:
    type: "constant"
    value: 50
```

---

## 条件评估器

### Q15: 条件判断总是失败

**原因**：占位符未正确替换或条件格式错误

**解决方案**：
1. 使用正确的占位符格式：

```yaml
conditions:
  - "%player_level% >= 10"  # PlaceholderAPI 格式
  - "{custom_var} == 100"   # 自定义变量格式
```

2. 确保传入占位符替换器：

```java
Function<String, String> replacer = text -> 
    PlaceholderAPI.setPlaceholders(player, text);
```

---

### Q16: 如何调试条件判断？

**解决方案**：

启用调试模式并记录结果：

```java
for (String condition : conditions) {
    Boolean result = ConditionEvaluator.evaluateSingle(condition, replacer);
    plugin.getLogger().info("条件 '" + condition + "' 结果: " + result);
}
```

## 性能问题

### Q17: 服务器 TPS 下降

**原因**：频繁的 GUI 刷新或大量表达式计算

**解决方案**：
1. 减少不必要的 GUI 刷新
2. 缓存频繁使用的物品源和表达式结果
3. 异步处理耗时操作

```java
// 异步处理
Bukkit.getScheduler().runTaskAsynchronously(plugin, () -> {
    // 耗时操作
    Data data = database.query();
    
    // 回到主线程更新
    Bukkit.getScheduler().runTask(plugin, () -> {
        session.refresh();
    });
});
```

---

### Q18: 内存占用过高

**原因**：会话未正确清理或缓存过多

**解决方案**：
1. 在玩家退出时清理会话：

```java
@EventHandler
public void onPlayerQuit(PlayerQuitEvent event) {
    guiService.close(event.getPlayer().getUniqueId());
}
```

2. 使用弱引用或定期清理缓存

---

## 兼容性问题

### Q19: 与其他 GUI 插件冲突

**原因**：多个插件监听相同的库存事件

**解决方案**：
1. 检查事件优先级
2. 使用 `InventoryHolder` 标识自己的 GUI：

```java
if (inventory.getHolder() instanceof GuiSession) {
    // 这是我们的 GUI
}
```

---

### Q20: PlaceholderAPI 占位符不生效

**原因**：PlaceholderAPI 未安装或未注册

**解决方案**：
1. 确认 PlaceholderAPI 已安装
2. 在 `plugin.yml` 中声明软依赖：

```yaml
softdepend:
  - PlaceholderAPI
```

3. 使用前检查是否可用：

```java
if (Bukkit.getPluginManager().isPluginEnabled("PlaceholderAPI")) {
    text = PlaceholderAPI.setPlaceholders(player, text);
}
```

## 开发问题

### Q21: 如何在代码中获取 CoreLib 实例？

**解决方案**：

```java
EmakiCoreLibPlugin coreLib = EmakiCoreLibPlugin.getInstance();
if (coreLib == null) {
    // CoreLib 未加载
}
```

---

### Q22: 如何扩展 CoreLib 的功能？

**解决方案**：

1. 注册自定义物品源解析器：

```java
ItemSourceRegistry.system().registerParser(myParser);
```

2. 实现 GuiSessionHandler 自定义事件处理：

```java
public class MyHandler implements GuiSessionHandler {
    @Override
    public void onSlotClick(GuiSession session, InventoryClickEvent event, GuiTemplate.ResolvedSlot slot) {
        // 自定义逻辑
    }
}
```

---

**解决方案**：

1. 启用详细日志：

```java
if (plugin.getConfig().getBoolean("debug")) {
    plugin.getLogger().info("调试信息: " + data);
}
```

2. 使用断点调试 IDE

## 其他问题

### Q24: 如何更新 CoreLib？

**解决方案**：
1. 停止服务器
2. 替换 `plugins/Emaki_CoreLib.jar`
3. 启动服务器
4. 检查配置文件是否需要更新

---

### Q25: 如何完全卸载 CoreLib？

**解决方案**：
1. 确保没有其他插件依赖 CoreLib
2. 停止服务器
3. 删除 `plugins/Emaki_CoreLib.jar`
4. 删除 `plugins/Emaki_CoreLib/` 目录
5. 启动服务器
