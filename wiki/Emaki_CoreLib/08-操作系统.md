# Emaki CoreLib - 操作系统

## 概述

操作系统（Operation System）是 Emaki CoreLib 的核心模块之一，提供了一套统一的操作定义、注册和执行机制。该系统支持声明式配置、条件执行、延迟调度、模板复用等高级特性，让插件能够通过简单的配置文件实现复杂的业务逻辑。

---

## 核心概念

### 架构概览

```
OperationRegistry (操作注册表)
    │
    ├── Operation (操作接口)
    │       ├── id: 操作标识
    │       ├── category: 操作分类
    │       ├── parameters: 参数定义
    │       └── execute(): 执行逻辑
    │
    ├── OperationExecutor (执行器)
    │       ├── execute(): 单操作执行
    │       ├── executeAll(): 批量执行
    │       └── 模板展开
    │
    └── OperationLineParser (行解析器)
            └── 解析操作行文本
```

### 操作生命周期

```
解析配置 → 验证参数 → 解析占位符 → 条件检查 → 概率判断 → 延迟调度 → 执行操作 → 返回结果
```

## 操作行语法

### 基本格式

```
操作ID 参数名=参数值 参数名=参数值 ...
```

### 示例

```
send_message text="<green>欢迎来到服务器！</green>"
give_money amount=1000
play_sound sound=entity.experience_orb.pickup volume=1.0 pitch=1.2
```

### 引号规则

当参数值包含空格时，需要使用引号包裹：

```
send_message text="这是一条包含空格的消息"
send_title title="<gold>标题" subtitle="<gray>副标题"
```

支持的引号类型：
- 双引号：`"value"`
- 单引号：`'value'`

### 转义字符

在引号内可使用转义字符：

| 转义序列 | 说明 |
|---------|------|
| `\\` | 反斜杠 |
| `\"` | 双引号 |
| `\'` | 单引号 |
| `\n` | 换行符 |
| `\t` | 制表符 |

## 控制指令

控制指令以 `@` 开头，用于控制操作的执行行为。

### @if - 条件执行

只有当条件为真时才执行操作。

```
send_message text="<green>恭喜升级！</green>" @if="%player_level% >= 10"
```

条件语法参考 [条件评估器](./06-条件评估器.md)。

### @chance - 概率执行

按指定概率执行操作（0.0 - 1.0）。

```
give_money amount=1000 @chance=0.5
```

### @delay - 延迟执行

延迟指定时间后执行操作。

```
send_message text="<yellow>3秒后传送...</yellow>"
teleport world=world x=100 y=64 z=200 @delay=60t
```

时间格式：
- `t`：游戏刻（1秒 = 20刻）
- `s`：秒
- 无单位：默认为游戏刻

### @ignore_failure - 忽略失败

即使操作失败也继续执行后续操作。

```
give_money amount=1000 @ignore_failure
send_message text="奖励已发放"
```

### 组合使用

多个控制指令可以组合使用：

```
give_money amount=500 @if="%player_level% >= 5" @chance=0.3 @delay=20t
```

## OperationRegistry - 操作注册表

### 功能说明

`OperationRegistry` 是操作的中心注册表，负责：
- 管理所有已注册的操作
- 按ID或分类查询操作
- 支持动态注册和注销

### API 接口

#### register() - 注册操作

```java
public OperationResult register(Operation operation)
```

**参数**：
| 参数名 | 类型 | 说明 |
|--------|------|------|
| `operation` | Operation | 操作实例 |

**返回值**：`OperationResult` 注册结果

**示例**：

```java
OperationRegistry registry = new OperationRegistry();
OperationResult result = registry.register(new SendMessageOperation());

if (!result.success()) {
    plugin.getLogger().warning("注册失败: " + result.errorMessage());
}
```

---

#### get() - 获取操作

```java
public Operation get(String operationId)
```

**参数**：
| 参数名 | 类型 | 说明 |
|--------|------|------|
| `operationId` | String | 操作ID（不区分大小写） |

**返回值**：`Operation` 或 `null`

---

#### byCategory() - 按分类获取

```java
public List<Operation> byCategory(String category)
```

**参数**：
| 参数名 | 类型 | 说明 |
|--------|------|------|
| `category` | String | 分类名称 |

**返回值**：`List<Operation>`

---

#### unregister() - 注销操作

```java
public void unregister(String operationId)
```

## OperationExecutor - 操作执行器

### 功能说明

`OperationExecutor` 是操作执行的核心组件，负责：
- 解析和执行操作行
- 处理控制指令（条件、概率、延迟）
- 管理模板展开
- 提供异步执行支持

### 构造方法

```java
public OperationExecutor(
    Plugin plugin,
    OperationRegistry registry,
    OperationLineParser lineParser,
    PlaceholderRegistry placeholderRegistry,
    OperationTemplateRegistry templateRegistry
)
```

---

### execute() - 执行单个操作

```java
public CompletableFuture<OperationResult> execute(
    OperationContext context,
    String operationId,
    Map<String, String> arguments
)
```

**参数**：
| 参数名 | 类型 | 说明 |
|--------|------|------|
| `context` | OperationContext | 执行上下文 |
| `operationId` | String | 操作ID |
| `arguments` | Map\<String, String\> | 参数映射 |

**返回值**：`CompletableFuture<OperationResult>`

**示例**：

```java
OperationExecutor executor = new OperationExecutor(plugin, registry, lineParser, placeholderRegistry, templateRegistry);

OperationContext context = OperationContext.create(plugin, player, "reward", false, false);

CompletableFuture<OperationResult> future = executor.execute(
    context,
    "give_money",
    Map.of("amount", "1000")
);

future.thenAccept(result -> {
    if (result.success()) {
        plugin.getLogger().info("操作执行成功");
    } else {
        plugin.getLogger().warning("操作执行失败: " + result.errorMessage());
    }
});
```

---

### executeAll() - 批量执行操作

```java
public CompletableFuture<OperationBatchResult> executeAll(
    OperationContext context,
    List<String> lines,
    boolean stopOnFailure
)
```

**参数**：
| 参数名 | 类型 | 说明 |
|--------|------|------|
| `context` | OperationContext | 执行上下文 |
| `lines` | List\<String\> | 操作行列表 |
| `stopOnFailure` | boolean | 失败时是否停止 |

**返回值**：`CompletableFuture<OperationBatchResult>`

**示例**：

```java
List<String> operations = List.of(
    "send_message text=\"<green>奖励发放中...</green>\"",
    "give_money amount=1000",
    "play_sound sound=entity.experience_orb.pickup volume=1.0 pitch=1.2",
    "give_exp amount=100"
);

CompletableFuture<OperationBatchResult> future = executor.executeAll(context, operations, true);

future.thenAccept(batch -> {
    if (batch.success()) {
        plugin.getLogger().info("所有操作执行成功，共 " + batch.steps().size() + " 个");
    } else {
        OperationStepResult failure = batch.firstFailure();
        plugin.getLogger().warning("操作失败于第 " + failure.lineNumber() + " 行: " + failure.result().errorMessage());
    }
});
```

## OperationContext - 执行上下文

### 功能说明

`OperationContext` 封装了操作执行所需的所有上下文信息，包括：
- 源插件实例
- 目标玩家
- 执行阶段标识
- 占位符变量
- 自定义属性

### 创建上下文

```java
OperationContext context = OperationContext.create(
    plugin,     // 源插件
    player,     // 目标玩家
    "reward",   // 阶段标识
    false,      // 是否静默模式
    false       // 是否调试模式
);
```

### 链式构建

```java
OperationContext context = OperationContext.create(plugin, player, "forge", false, false)
    .withPlaceholder("item_name", "烈焰之剑")
    .withPlaceholder("quality", "传说")
    .withAttribute("custom_data", someObject);
```

### 访问器方法

| 方法 | 返回类型 | 说明 |
|------|---------|------|
| `sourcePlugin()` | Plugin | 源插件实例 |
| `player()` | Player | 目标玩家 |
| `phase()` | String | 执行阶段 |
| `silent()` | boolean | 是否静默模式 |
| `debug()` | boolean | 是否调试模式 |
| `placeholders()` | Map\<String, String\> | 占位符映射 |
| `attributes()` | Map\<String, Object\> | 属性映射 |
| `placeholder(key)` | String | 获取占位符值 |
| `attribute(key)` | Object | 获取属性值 |

## 操作模板

### 配置模板

在 `config.yml` 中定义操作模板：

```yaml
operation:
  debug: false
  templates:
    forge_success_common:
      - 'send_message text="<green>锻造成功！获得了 %forge_result_item_name%</green>"'
      - 'play_sound sound=entity.experience_orb.pickup volume=0.8 pitch=1.2'
      - 'give_exp amount=50'
    
    forge_success_legendary:
      - 'send_title title="<gold>传说品质！" subtitle="<gray>恭喜获得传说装备"'
      - 'broadcast_message text="<gold>%player_name% 锻造出了传说品质的 %forge_result_item_name%！"'
      - 'play_sound sound=ui.toast.challenge_complete volume=1.0 pitch=1.0'
      - 'spawn_particle type=totem_of_undying count=50 offset=2.0'
      - '@delay=20t give_exp amount=200'
```

### 使用模板

```java
List<String> operations = List.of(
    "use_template name=forge_success_legendary with.item_name=\"烈焰之剑\""
);

executor.executeAll(context, operations, true);
```

### 模板变量

模板变量通过 `with.` 前缀传递：

```
use_template name=my_template with.variable1="值1" with.variable2="值2"
```

在模板内部，变量以 `template_` 为前缀访问：

```yaml
templates:
  my_template:
    - 'send_message text="变量1: {template_variable1}, 变量2: {template_variable2}"'
```

## 自定义操作

### 实现操作接口

```java
public class CustomOperation implements Operation {
    
    @Override
    public String id() {
        return "custom_action";
    }
    
    @Override
    public String category() {
        return "custom";
    }
    
    @Override
    public String description() {
        return "自定义操作描述";
    }
    
    @Override
    public List<OperationParameter> parameters() {
        return List.of(
            OperationParameter.required("value", OperationParameterType.STRING, "参数说明"),
            OperationParameter.optional("option", OperationParameterType.INTEGER, "10", "可选参数")
        );
    }
    
    @Override
    public OperationResult execute(OperationContext context, Map<String, String> arguments) {
        String value = arguments.get("value");
        int option = OperationParsers.parseInt(arguments.get("option"), 10);
        
        if (context.player() == null) {
            return OperationResult.failure(OperationErrorType.INVALID_STATE, "需要玩家上下文");
        }
        
        // 执行自定义逻辑
        doSomething(context.player(), value, option);
        
        return OperationResult.ok();
    }
}
```

### 继承 BaseOperation

```java
public class SimpleOperation extends BaseOperation {
    
    public SimpleOperation() {
        super(
            "simple_action",
            "utility",
            "简单操作示例",
            OperationParameter.required("input", OperationParameterType.STRING, "输入值")
        );
    }
    
    @Override
    public OperationResult execute(OperationContext context, Map<String, String> arguments) {
        OperationResult playerCheck = requirePlayerResult(context);
        if (!playerCheck.success()) {
            return playerCheck;
        }
        
        String input = stringArg(arguments, "input");
        // 执行逻辑...
        
        return OperationResult.ok();
    }
}
```

### 注册自定义操作

```java
OperationRegistry registry = new OperationRegistry();
registry.register(new CustomOperation());
registry.register(new SimpleOperation());
```

## 最佳实践

### 1. 使用模板复用逻辑

将重复的操作序列定义为模板：

```yaml
templates:
  notify_admin:
    - 'run_command_as_console command="say [通知] %message%"'
```

### 2. 合理使用控制指令

```yaml
operations:
  - 'give_money amount=1000 @if="%player_level% >= 10"'
  - 'send_message text="<red>金币不足</red>" @if="%player_balance% < 100" @ignore_failure'
```

### 3. 异步执行耗时操作

```java
executor.executeAll(context, operations, true)
    .thenAccept(batch -> {
        // 异步回调处理
    });
```

### 4. 调试模式

启用调试模式查看执行日志：

```yaml
operation:
  debug: true
```
